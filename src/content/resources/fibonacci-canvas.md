## El Anti-Patrón del "Big Bang"

El 68% de proyectos de transformación digital fallan. No por mala tecnología, sino por saltar directo de idea a ejecución masiva sin validar supuestos.

El Fibonacci Canvas te fuerza a crecer orgánicamente: cada fase se construye sobre aprendizajes reales de las anteriores.

## Estructura del Canvas

### Fase 0: Overview (Contexto)
**Duración típica: 1-2 semanas**

**Preguntas clave:**
- ¿Qué problema resuelves?
- ¿Para quién?
- ¿Cómo se mide el éxito?
- ¿Cuál es el costo de no hacer nada?

**Output esperado:**  
Una frase clara: "Reducir X en Y%, impactando Z"

Ejemplo: "Reducir tiempo de onboarding en 80%, impactando 500 clientes/mes y $480K anuales"

### Fase 1: Discovery (Descubrimiento)
**Duración típica: 4-6 semanas**  
**Inversión: φ¹ relativa a Phase 0**

**Actividades:**
- [ ] Entrevistas con stakeholders (mín. 8 personas)
- [ ] Análisis de datos históricos (mín. 6 meses)
- [ ] Mapeo de procesos actuales (as-is)
- [ ] Identificación de fricciones (top 10)
- [ ] Benchmarking de industria

**Output esperado:**  
Documento de diagnóstico con:
- Fricciones priorizadas por impacto
- Quick wins (0-3 meses)
- Medium wins (3-6 meses)
- Long term (6+ meses)

**Gate para pasar a Strategy:**  
✅ Al menos 3 fricciones cuantificadas con $impacto anual

### Fase 2: Strategy (Estrategia)
**Duración típica: 4-8 semanas**  
**Inversión: φ² relativa**

**Actividades:**
- [ ] Diseño de arquitectura de solución
- [ ] Modelado financiero (ROI, payback, NPV)
- [ ] Análisis de riesgos y mitigaciones
- [ ] Roadmap en sprints con hitos medibles
- [ ] Definición de KPIs y dashboard

**Output esperado:**  
Propuesta ejecutiva con:
- Arquitectura técnica (diagramas)
- Modelo financiero (3 escenarios)
- Plan de implementación (Gantt)
- Métricas de éxito (OKRs)

**Gate para pasar a Execution:**  
✅ ROI proyectado >200% en 18 meses  
✅ Aprobación de presupuesto  
✅ Equipo asignado

### Fase 3: Execution (Ejecución)
**Duración típica: 8-16 semanas**  
**Inversión: φ³ relativa**

**Actividades:**
- [ ] Sprint 0: Setup de ambiente y equipo
- [ ] MVP en piloto controlado (1-5% de alcance)
- [ ] Validación con datos reales (A/B test si aplica)
- [ ] Iteración basada en feedback
- [ ] Documentación y training

**Output esperado:**  
Sistema funcional en piloto con:
- KPIs alcanzados (sí/no para cada uno)
- Learnings documentados
- Plan de rollout a escala

**Gate para pasar a Scale:**  
✅ Piloto demuestra >80% de ROI proyectado  
✅ NPS de usuarios piloto >7/10  
✅ Sistema opera sin intervención diaria del equipo dev

### Fase 4: Scale (Escala)
**Duración típica: 12-24 semanas**  
**Inversión: φ⁴ relativa**

**Actividades:**
- [ ] Rollout incremental (ej: 10% → 50% → 100%)
- [ ] Monitoreo de KPIs en tiempo real
- [ ] Ajustes y optimización continua
- [ ] Capacitación masiva de usuarios
- [ ] Transición a operación estándar

**Output esperado:**  
Sistema en producción completa:
- 100% de usuarios migrados
- Ahorros/ingresos materializados
- Equipo interno capacitado para operar
- Documentación completa

**Gate para cerrar proyecto:**  
✅ ROI real dentro de 20% del proyectado  
✅ Sistema autosostenible (no requiere equipo externo)  
✅ Retrospectiva documentada

## Cómo Usar el Canvas

### Para Proyectos Nuevos
1. Imprime el canvas en A1 (o usa herramienta digital)
2. Sesión de 2 horas con stakeholders clave
3. Llena Phase 0 y 1 en la sesión
4. Asigna dueños a cada fase
5. Revisa semanalmente en stand-up

### Para Proyectos En Curso
1. Mapea dónde estás hoy (¿completaste realmente cada fase?)
2. Identifica gates que saltaste (riesgo oculto)
3. Decide: ¿Regresar a validar o continuar con mitigación?

## Casos de Uso

### ✅ Perfecto Para
- Proyectos de transformación digital ($100K+)
- Iniciativas multi-equipo (>5 personas)
- Cuando el costo de fallar es alto
- Venta a C-level (el canvas es tu pitch)

### ⚠️ Overkill Para
- Proyectos <$20K
- Iniciativas exploratoria (R&D puro)
- Proyectos con deadline inflexible (ej: regulatorio)
